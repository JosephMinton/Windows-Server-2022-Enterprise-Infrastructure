<h1><a href="https://www.linkedin.com/in/josephkdminton/">Joseph Minton</a>'s Enterprise Active Directory Security Governance Lab 🖥</h1>

This project simulates an enterprise Windows domain environment. The project focuses on building foundational infrastructure and implementing centralized security governance using Active Directory and Group Policy.

The objective is to demonstrate practical competency in Windows Server administration, identity management, and security enforcement within a corporate network model.

- [Active Directory Foundations](https://github.com/JosephMinton/Windows-Server-2022-Enterprise-Infrastructure/blob/main/Active%20Directory%20Domain%20Services%20Foundation.md) Deployed Windows Server 2022 domain environment within VMware. Created GPOs, OUs, groups, and users. It explains group scopes and types, demonstrates how to create groups, and includes a practical exercise to implement these concepts in a basic AD environment.
- [Security Governance with Group Policy](https://github.com/JosephMinton/Windows-Server-2022-Enterprise-Infrastructure/blob/main/Security%20Governance%20via%20Group%20Policy%20Management.md) Implemented centralized administrative control using Group Policy to enforce password policies, system restrictions, and enterprise wide configuration standards across the domain.
- [Domain Integration & Security Policy Verification](https://github.com/JosephMinton/Enterprise-Active-Directory-Security-Governance-Lab/blob/main/Domain%20Integration%20%26%20Security%20Policy%20Verification.md) Integrated a Windows 11 client into the domain by configuring static network settings. Verified that administrative controls were enforced by troubleshooting with PowerShell, ensuring security policies functioned as intended on domain joined systems.
- [Centralized Network Sharing and FSRM Implementation](https://github.com/JosephMinton/Enterprise-Active-Directory-Security-Governance-Lab/blob/main/File%20Services%2C%20Resource%20Management%2C%20and%20Data%20Governance.md) establishing centralized file sharing through the configuration of NTFS and share permissions, alongside automating access via GPO driven network drive mapping. It also involves implementing File Server Resource Manager (FSRM) to manage storage quotas and file screening, while utilizing Access-Based Enumeration (ABE) to hide sensitive folders from unauthorized users.
